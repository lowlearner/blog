2、内存查看

1、字符串内存布局

  • ALT+6打开内存窗口
  • 将字符串拖入或者输入字符串可查看内存布局
  • 宽字符（UNICODE编码）：每个字符占2个字节。 UTF-8中文字符3个字节，英文1个字节

2、整型变量内存布局

  • 整型变量内存布局涉及字节序。
  • PowerPC系列采用大端存储方式， X86方式采用小端存储方式。

3.调试执行

  • 启动调试 F5
  • 逐语句执行 F11 （进入函数）
  • 逐过程执行 F10 （不进入函数）
  • 跳出执行 Shift+F11
  • 运行到光标处 Ctrl+F10
  • 多次执行代码 ：移动执行指针 （1、不要移动到函数外； 2、不要跳过重要的初始化操作语句；）

2、断点管理

  • 设置 （break or b)（rb or rbreak 正则表达式断点）（tb or tbreak 临时断点）

    • break 文件名：行号
    • break 函数名
    • rb 正则表达式
    • b +/- 偏移量
    • b 断点 条件
    • b * 指令地址
    • tb 断点

  • 启用/禁用

    • disable 断点编号
    • enable 断点编号
    • enable once 断点编号
    • enable delete 断点编号
    • enable count 数量 断点编号
    • ignore 断点编号 次数

  • 查看断点

    • info / i breakpoints / break / b

  • 删除

    • delete 断点编号或范围
    • clear 函数名
    • clear 行号

  • 启动程序

    • run / r

  • 继续运行

    • continue / c
    • continue 次数
    • finish 继续运行至当前函数执行完成

  • 单步执行

    • step / s

  • 逐过程

    • next / n

  • 查看函数参数

    • info / i args

  • 查看/修改变量

    • print / p 变量名
    • 可使用gdb内嵌函数
    • p *指针
    • set print null-stop （删除空字符）遇到结束字符停止显示
    • set print pretty(漂亮)
    • set print array on 查看数组

  • 自动显示

    • display 变量名
    • undisplay ***
    • delete display numID
    • disable display numID
    • enable display numID

  • 显示源码

    • list / l ***
    • set listsize num

发行（release）版调试

gdb

  • 从调试版中提取调试符号
    • 生成调试符号表： objcopy --only-keep-debug 程序名 ***.symbol
    • 加上调试符号调试发行版： gdb --symbol=***.symbol -exec=***_release
  • 直接使用调试版作为符号源
    • gdb --symbol=*** -exec=***_release
