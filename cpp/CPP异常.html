异常

什么是异常？

可以“向上抛出”调用层次结构的对象

  • throwing 将控制权转移会当前函数的调用者
  • 通过 块 捕获处理 try ... catch
  • 如果不处理，异常会传播直到到main
  • 如果在main中没有处理， std::terminate将被调用
  • std::terminate默认终止程序

Usages: Report Contract Violations

前提条件违规

  • 前提条件 = 关于有效函数参数的期望
  • 违规示例：越界容器索引
  • 检查输入前提条件的函数称为宽契约
  • 由于总是会进行昂贵的额外有效性检查（即使已知传入的参数是有效的），宽合约函数在很大程度上已经失宠

未能建立/保留不变量

  • 公共成员函数无法设置有效的成员值
  • 示例：向量增长期间内存不足

后置条件违规

  • 后置条件 = 对输出的期望（返回值）
  • 违规 = 函数未能产生有效的返回值
  • 示例：构造函数失败，无法返回除以零的结果

优点缺点

  • 将错误处理代码与业务逻辑分离
  • 错误处理的集中化（调用链的上层）
  • 现在不抛出异常时对性能的影响可以忽略不计
  • 但是，通常对性能的影响时异常被抛出
  • 由于额外的有效性检查而导致的性能影响
  • 容易产生资源泄漏（更多见下文）

异常的替代方案

如果输入值无效（违反前提条件）

  • 窄合约函数：在传递参数之前确保参数有效
  • 使用排除无效值的参数类型
  • 为了更好的性能，这是现在的首选

如果未能建立/保留不变量

  • 错误状态/标志
  • 将对象设置为特殊的、无效的值/状态

如果无法返回有效值（后置条件违规）

  • 通过单独的输出参数（引用或指针）返回错误代码
  • 返回特殊的无效值
  • 使用可以包含有效结果或不包含任何结果的特殊词汇类型 ，如 C++17std::optional或 Haskell‘ Maybe

Problems & Guarantees

Resource Leaks

几乎任何一段代码都可能抛出异常

⇒ 对 C++ 类型和库的设计影响很大

微妙的资源/内存泄漏的潜在来源，如果使用
  • 执行自己的内存管理的外部 C 库
  • （设计不佳）不使用 RAII 进行自动资源管理的 C++ 库
  • （设计不佳）在销毁时不清理资源的类型
示例：由于 C 风格的资源处理造成的泄漏

即资源初始化（connect）和终结（disconnect）两个独立的函数

使用 RAII 防止泄漏！

RAII又是什么？
  • 构造函数：资源获取
  • 析构函数：资源释放/终结
如果抛出异常
  • 本地范围内的对象被销毁：调用的析构函数
  • 与 RAII：资源正确释放/最终确定

如果您必须使用一个库（例如，来自 C 的库），该库使用单独的函数来初始化和终止资源，请编写一个 RAII 包装器。

通常，使您的包装器不可复制（删除复制构造函数和复制赋值运算符）也是有意义的，尤其是在无法控制引用的外部资源的情况下。

例外保证

无保证

  • 操作可能会失败
  • 资源可能会泄露
  • 可能违反不变量（= 成员可能包含无效值）
  • 部分执行失败的操作可能会导致副作用（例如输出）
  • 异常可能向外传播

基本保证

  • 不变量被保留，没有资源泄漏
  • 所有成员都将包含有效值
  • 部分执行失败的操作可能会导致副作用（例如，值可能已写入文件）

强保证（提交或回滚语义）

  • 操作可能会失败，但不会有明显的副作用
  • 所有成员保留其原始值

不投掷保证（最强）

  • 保证操作成功
  • 从外部无法观察到的异常（在内部没有抛出或捕获）

No-Throw Guarantee: noexcept C++11

  • 'foo' 承诺永远不会抛出异常/让任何逃脱
  • 如果异常从 noexcept 函数中逃脱 ⇒ 程序将被终止
仔细想想你是否能遵守No-Throw的承诺！
  • noexcept 是函数接口的一部分（甚至是 C++17 中函数类型的一部分）
  • 稍后将 noexcept 函数改回抛出函数可能会破坏依赖于不必处理异常的调用代码

Conditional noexcept

A **noexcept(**expression )

declares ‘A’ as noexcept if expression yields true

A noexcept( noexcept(B) )

declares ‘A’ as noexcept if ‘B’ is noexcept

noexcept(*true*) by Default

是隐式声明的特殊成员
  • 默认构造函数
  • 析构函数
  • 复制构造函数，移动构造函数
  • 复制赋值运算符、移动赋值运算符
  • 继承构造函数
  • 用户定义的析构函数
除非
  • 他们需要调用一个 noexcept(false) 的函数
  • 明确声明另有说明
